# Kaivue Video Wall

Native Qt 6 / C++ desktop client for Security Operations Center (SOC) operators
running the Kaivue Recording Server. Drives 4-32 monitors with 64+ concurrent
hardware-decoded streams, PTZ keyboard/joystick hardware, and operator
workflows (alert triage, incident escalation, shift handover).

> **Wave 1 scaffold only.** This commit lands the empty Qt Quick window plus
> the CMake + vcpkg build system. The actual streaming pipeline (WebRTC, HLS,
> RTSP, hardware decode) ships in **Wave 2+**.

## Why native (not Flutter)

- 64+ concurrent live streams with hardware decode is beyond Flutter's ceiling.
- Multi-monitor support (8+ displays per workstation) needs native desktop APIs.
- PTZ hardware (Axis T8311, Axiom, Honeywell, Bosch) needs USB HID + serial.
- Every competitor (Genetec, Milestone, Avigilon) ships on Qt for the same reasons.

## Prerequisites

- **CMake 3.24** or newer
- **vcpkg** (bootstrap below)
- A Qt 6.5+ compatible C++20 toolchain:
  - Windows: **Visual Studio 2022** (MSVC v143 + Windows 10/11 SDK)
  - Linux: **GCC 12** or newer (Ubuntu 22.04: `sudo apt install build-essential`)
- Git

## Bootstrap vcpkg

```bash
git clone https://github.com/microsoft/vcpkg.git ~/vcpkg
cd ~/vcpkg
./bootstrap-vcpkg.sh    # Linux / macOS
.\bootstrap-vcpkg.bat   # Windows
export VCPKG_ROOT="$HOME/vcpkg"
```

On Windows (PowerShell):

```powershell
$env:VCPKG_ROOT = "$HOME\vcpkg"
```

vcpkg will download Qt 6 from source on first build (this takes a while —
expect 30-90 minutes depending on hardware). Subsequent builds are cached.

## Build — Linux (Ubuntu 22.04, GCC 12)

```bash
sudo apt install build-essential cmake ninja-build pkg-config \
    libgl1-mesa-dev libxkbcommon-dev libfontconfig1-dev \
    '^libxcb.*-dev' libx11-xcb-dev libglu1-mesa-dev

cd video-wall
cmake -B build -G Ninja \
    -DCMAKE_TOOLCHAIN_FILE="$VCPKG_ROOT/scripts/buildsystems/vcpkg.cmake" \
    -DCMAKE_BUILD_TYPE=Release
cmake --build build
./build/KaivueVideoWall
```

## Build — Windows (MSVC 2022)

From a "x64 Native Tools Command Prompt for VS 2022":

```powershell
cd video-wall
cmake -B build -G "Visual Studio 17 2022" -A x64 `
    -DCMAKE_TOOLCHAIN_FILE="$env:VCPKG_ROOT\scripts\buildsystems\vcpkg.cmake"
cmake --build build --config Release
.\build\Release\KaivueVideoWall.exe
```

## Distribution (deferred)

- **Windows installer**: Qt Installer Framework, **EV code signed**. Hooked up
  at the end of the roadmap — EV cert procurement is the long pole.
- **Linux**: `.deb` and `.rpm` packages. Wired up alongside the Windows installer.
- **macOS**: deferred.

`CMakeLists.txt` already includes stubbed `install()` rules and a
`qt_generate_deploy_qml_app_script()` invocation so the installer step can
slot in without restructuring the build.

## Performance invariants (target — Wave 2+)

- **64 concurrent 1080p streams** on RTX 4060-class hardware with **P99 frame time < 16 ms**.
- **Cold start to first frame < 2 s**.
- **Scene switch < 500 ms**.
- Per-stream quality auto-adjusts based on monitor pixel coverage.
